# 食用指南

## 一、注册 Cloudinary 账号

1. 访问 [Cloudinary](https://cloudinary.com)
2. 点击 **Sign Up for Free** 注册免费账号
3. 注册完成后会进入控制台，无需付费（免费套餐每月 25GB 流量）

---

## 二、配置 Fetch 白名单

Cloudinary Fetch 需要在控制台添加允许的源站域名，否则无法代理图片。

1. 登录 [Cloudinary Console](https://cloudinary.com/console)
2. 点击左侧 **Settings**（齿轮图标）→ **Security**
3. 滚动到 **Allowed fetch domains** 部分
4. 点击 **Add domain**，添加以下域名：
   ```
   himawari.asia
   ```
5. 点击 **Save** 保存

---

## 三、获取 Cloud Name

1. 登录 Cloudinary 控制台
2. 在 **Dashboard** 首页即可看到 **Cloud Name**（例如 `abc123xyz`）
3. 复制该值备用

---

## 四、在 App 中配置

1. 安装并打开馒头地球
2. 长按壁纸进入 **高级设置**
3. 找到 **Cloudinary CDN** 分类
4. 在 **Cloud Name** 输入框中填入你的 Cloud Name
5. 返回即可，App 会自动通过 CDN 获取卫星图像

---

## 五、架构说明

```
向日葵卫星源站 (himawari.asia)
         ↓ Cloudinary Fetch 代理
   Cloudinary CDN 缓存
         ↓
      你的设备
```

- 本应用**始终使用 Cloudinary CDN** 获取图像（无直连源站选项）
- 源站：[`himawari.asia`](https://himawari.asia)（NICT 科学云项目，URL 格式兼容旧版 himawari8）
- 时间戳：通过 `himawari.asia/img/D531106/latest.json` API 实时获取最新可用帧，确保获取到的是实际存在的图像
- **无需 API Key / Secret**：仅使用 Cloudinary Fetch 代理功能，纯读取模式
- **自动图片优化**：CDN 自动进行格式转换（WebP/AVIF）和压缩，节省流量

## 六、更新机制说明

### 更新间隔

在壁纸主界面可通过滑块设置更新间隔，范围 **10–120 分钟**（步长 10 分钟）。设置完成后，App 会按该间隔定期请求最新图像。

### 两种更新模式

在高级设置顶部的「更新方式」中可选择：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **SyncAdapter**（默认）| 由 Android 系统调度，更省电 | 大多数设备推荐 |
| **Background Service** | 前台常驻服务，更新更准时 | 部分系统（如某些国产 ROM）系统调度不可靠时使用 |

> **提示**：两种模式均会读取你设置的更新间隔。如使用 Background Service 模式，通知栏会显示「馒头地球正在后台运行」。

### 缓存策略

- 图像按帧缓存（每 10 分钟一帧），48 小时内同一帧不重复下载
- 缓存满 48 小时后自动清理旧文件

## 七、常见问题

### Q: 不填 Cloud Name 会怎样？

A: App 无法通过 CDN 获取图像，壁纸将不会更新。同步时日志中会提示 `cloud_name is not configured`。

### Q: Cloud Name 填错了会怎样？

A: 图片加载会失败。如使用 Background Service 模式，通知栏会显示 `"Last update: HH:MM (failed)"` 提示失败状态。请检查是否正确复制了控制台首页的 Cloud Name。

### Q: 免费套餐够用吗？

A: 免费套餐每月 25GB 流量。每次更新约消耗几百 KB（分辨率越高越大），正常使用绰绰有余。

### Q: 如何验证 CDN 是否生效？

A: 开启调试模式（高级设置 → 调试模式），壁纸上会显示抓取状态和调试信息。也可在 Logcat 中过滤 `EarthFetcher` 或 `WallpaperDebug` 标签查看详细日志。

### Q: 设置了 30 分钟间隔，但实际更新时间不准？

A: SyncAdapter 模式依赖 Android 系统调度，Doze 省电策略可能导致延迟，属正常现象。如需更准时的更新，可切换到 Background Service 模式。

### Q: 为什么有时壁纸长时间不更新？

A: 可能原因：
1. Cloud Name 未填写或填写错误
2. 网络不可用或 Cloudinary 服务暂时异常
3. 节能网络下（系统判定为计量网络），App 会自动降为 120 分钟间隔并降低分辨率
4. 向日葵卫星源站本身维护，`latest.json` 暂未更新
